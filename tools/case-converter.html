camelCase vs PascalCase vs snake_case — When to Use Each

Different programming languages and contexts have different conventions. Using the wrong case in code causes naming inconsistencies that make codebases harder to read and maintain.

  • camelCase — JavaScript variables, function names, JSON keys, Swift/Kotlin variables
  • PascalCase — Class names in any OOP language, React components, TypeScript interfaces
  • snake_case — Python variables and functions, Ruby conventions, SQL column names
  • kebab-case — CSS class names, HTML custom attributes, URL slugs, npm package names
  • UPPERCASE — Constants in JavaScript/Python (MAX_RETRIES), environment variables

Title Case vs Sentence Case — Content Writing

Title Case capitalizes the first letter of every word. It is the standard for article titles, product names, and formal headings. Sentence case capitalizes only the first word and proper nouns — it is the modern standard for UI labels, button text, and subheadings in design systems like Google Material and Apple HIG.
