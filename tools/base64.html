What is Base64 Encoding?

Base64 is a method of encoding binary data into a string using only 64 printable ASCII characters — the letters A–Z and a–z, digits 0–9, plus + and /. The name "Base64" comes from this 64-character alphabet. It was designed so that binary data could be safely transmitted through systems that only handle text — like email protocols, JSON APIs, HTML attributes, and XML documents — without the raw binary bytes corrupting the format.

Every 3 bytes of input data become 4 Base64 characters. This means Base64-encoded output is always approximately 33% larger than the original input. The = or == padding at the end is added when the input length is not divisible by 3.

Base64 vs Base64URL — What is the Difference?

Standard Base64 uses + and / characters, which are special characters in URLs and HTTP headers. Base64URL is a variant that replaces + with - and / with _, making it safe for URL query strings and HTTP headers without percent-encoding.

JWT (JSON Web Tokens) use Base64URL encoding — not standard Base64. To decode a JWT payload with this tool: take the middle section (between the two dots), replace all - with + and _ with /, switch to Decode mode, and paste it in.

Is Base64 the Same as Encryption?

No — and this is a very common misconception. Base64 is encoding, not encryption. Anyone can decode a Base64 string instantly without a password or key. It does not protect your data — it only makes binary data text-safe for transmission. If you need to protect sensitive data, use proper encryption like AES-256 or bcrypt. Never store passwords, API secrets, or private keys as Base64 and assume they are secure.

How to Decode a JWT Token

A JWT consists of three Base64URL-encoded parts separated by dots: header.payload.signature. To read the payload claims:

  • Copy the middle section of your JWT (between the first and second dot)
  • Replace all - characters with +
  • Replace all _ characters with /
  • Switch to Decode mode in the tool above and paste it in
  • You will see the raw JSON with user ID, email, expiry (exp), and any custom claims
