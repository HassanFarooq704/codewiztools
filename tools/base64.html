What Is Base64 Encoding?

Base64 is a binary-to-text encoding scheme that converts data into a string of 64 ASCII characters (A–Z, a–z, 0–9, +, /). The "64" refers to the 64 printable ASCII characters used. It was designed to allow binary data to be safely embedded in text-only contexts like email, HTML, JSON, or XML — where raw binary bytes would otherwise corrupt the format.

When Should You Use Base64?

  • Decoding JWT (JSON Web Token) payloads to inspect claims
  • Creating data: URIs to embed images inline in HTML or CSS
  • Encoding HTTP Basic Authentication credentials for API testing
  • Inspecting Kubernetes secrets or Docker config.json files
  • Passing binary data in JSON REST API request bodies
  • Encoding email attachments in MIME message format

Base64 vs Base64URL

Standard Base64 uses + and / characters which are unsafe in URLs. Base64URL replaces + with - and / with _, making it safe for URL query parameters and HTTP headers. JWT tokens use Base64URL encoding. This tool uses standard Base64 which is correct for most use cases including data URIs, MIME, and general string encoding.
