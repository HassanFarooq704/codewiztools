What is URL Encoding (Percent Encoding)?

URL encoding — officially called percent encoding — converts characters that are not allowed in a URL into a safe, universally transmittable format. Every unsafe character is replaced with a % sign followed by its two-digit hexadecimal ASCII code. For example, a space becomes %20, an ampersand becomes %26, and a forward slash becomes %2F.

This ensures that URLs transmit without ambiguity across HTTP, APIs, browsers, and servers regardless of the character set or operating system. The standard is defined in RFC 3986 and is used everywhere — from Google search queries to REST API calls.

Percent Encoding Converter — encodeURI vs encodeURIComponent

This tool supports both JavaScript encoding functions. Use the Method selector in the tool controls to switch between them:

  • encodeURIComponent — Encodes everything except letters, digits, and - _ . ! ~ * ' ( ). Use this for individual query parameter values inside a URL. This is the correct choice for most use cases.
  • encodeURI (full URL) — Encodes everything except characters that have structural meaning in URLs (: / ? # [ ] @ ! $ & ' ( ) * + , ; =). Use this when encoding a complete URL and you want to preserve its structure.

If you are passing a value inside a query string — for example, a search term or a redirect URL — always use encodeURIComponent. If you are encoding a full URL to store or transmit as a string, use encodeURI.

What characters need to be percent encoded in a URL?

Any character outside the unreserved set must be encoded when used as data in a URL. The unreserved characters that are always safe are: letters A–Z, a–z, digits 0–9, and the symbols - _ . ~. Everything else — spaces, punctuation, non-ASCII scripts — must be percent encoded. See the reference table above for the most common encodings.

How to URL encode non-English text (Arabic, Urdu, Unicode)

This tool supports full Unicode. Non-ASCII text like Arabic, Urdu, Chinese, Japanese, or emoji is first converted to UTF-8 bytes, and then each byte is percent-encoded. For example, the Urdu word "اردو" encodes to %D8%A7%D8%B1%D8%AF%D9%88. This is the correct and modern standard per RFC 3986.
